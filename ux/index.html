Treehouse User Experience

Thoughtful details are the secret to making an experience successful and desirable.

  • Goals
  •  Make Treehouse easier to use
  • Measuring success
  •  Use existing metrics tracking student satisfaction

Using Treehouse feels good every time because of its thoughtful design. From the Home dashboard to the Community section, it's one coherent, seamless experience. Product design is a never-ending task, and there's always opportunities to make something great even better. Here are some design opportunities I've identified.

Making video transcripts clickable

As an experienced programmer, I use Treehouse to learn specific technologies and refresh my knowledge of certain areas. One feature that I find particularly useful is being able to quickly glance at the whole video transcript of a lesson. It allows me to know if I can skip ahead to a certain point.

I would love being able to just click on a line of text and be taken to that specific section of the video. It's a minor detail, but one that could improve the experience of power users significantly. It's probably easy to implement by setting the currentTime property of HTML5 video using vanilla JavaScript.

Making giving feedback about code in the Community even easier

If a code-related question is being asked in the Community, it's highly likely that the answers will feature a modified version of that code. We could try including it automatically in the answer. This is a change that can easily be validated by A/B testing, or by interviewing a subset of Treehouse's great mods to see how they feel about it.

Also, if a question is tagged "Swift", all code that references it can probably be syntax-highlighted with this language's rules. The image above shows an answer by me on one occasion I forgot to use the ```swift special syntax.

Investigating why many students post duplicate questions

I think the way both Treehouse and Quora try to prevent duplicate questions by providing search results right when the user intends to post is very clever. However, there seems to be some students that are posting the exact same question with seconds of difference. Is this a user slip, a user mistake, or a system error? Can we help prevent it?

  • Designing, prototyping and A/B testing subtle changes like these.
  • Performing heuristic evaluations on the system to uncover more ways we could make it more usable.
